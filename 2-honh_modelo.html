# Vamos criar uma funo que treino e traz a avaliao do modelo

def train_and_evaluate_model(model, model_name):
    print(f"Treinando e avaliando: {model_name}")
    
    # Treinamento do modelo
    model.fit(X_train, y_train)
    
    # Avaliao com ClassificationReport do Yellowbricks
    visualizer = ClassificationReport(model, classes=['Aliengena', 'Humano'], support=True)
    visualizer.score(X_test, y_test)
    visualizer.poof(outpath=f"{model_name}_classification_report.png") # salva as avaliaes para melhor comparao
    
    # Exibir o grfico
    plt.show()

Mtricas de avaliao

  1. Precision
    • Proporo de verdadeiros positivos (TP) sobre o total de positivos preditos (TP+FP)
  2. Recall (sensibilidade)
    • Proporo de verdadeiros positivos (TP) sobre o total de positivos reais (TP+FN)
  3. F1 Score
    • a mdia harmnica entre Precision e Recall
  4. Support
    • Mostra quantos exemplos verdadeiros pertencem a cada classe

Vemos que a classe dos Alien so mais difceis de prever pelo modelo, o que faz sentido devido a variabilidade das suas caractersticas.Temos ETEzes bem diferentes no mundo de Star Wars, de um Wookiee a um Ewok. A nica mtrica em que foi mais alto nos Aliens o recall que indica menos falsos negativos. Ele acerta bem em dizer que a observao no um Alien.

Bom, a mtrica que vamos prestar mais ateno o F1 Score, j que o modelo mais para brincadeira identificando a classe pelas caractersticas. No precisamos tomar tanto cuidado com falso negativo ou positivo e sim ter um modelo que acerta mais no geral.

No vamos olhar para acurcia, j que uma medida em que mostra Nmero de previses corretas no total das previses. Mas uma medida com muitas limitaes e acredito que ser muito enviesada com um conjunto de dados com poucas observaes assim como o que estamos analisando.

Grid Search

  • Agora vamos deixar o grid search descobrir os melhores hiperparmetros
  • Vamos deixar o grid search descobrir um hiperparmetro por vez, entre:
  1. n_neighbors (N vizinhos)
    • Esse hiperparmetro a quantidade de vizinhos a ser considerado para determinar a classe da observao que est sendo avaliada.
  2. weights (Pesos)
    • Muda o peso atribudo aos vizinhos:
      • uniforme: todos os vizinhos tem o mesmo peso.
      • distance: vizinhos mais prximos tem maior peso.
  3. metric (Distncia)
    • Altera a forma de calcular a distncia:
      • Euclidiana: uma linha reta entre os pontos.
      • Mangattan: como se andasse por quarteres at alcanam o outro ponto.
